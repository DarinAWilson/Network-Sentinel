# Network Sentinel Architecture

![Network Sentinel Architecture](../Diagrams/Architecture-Diagram-v2.png)

## Overview

Network Sentinel is a Dockerized network security monitoring platform designed to help small businesses and advanced home users monitor network activity, collect security events, visualize system information, and better understand security alerts.

The platform combines intrusion detection, centralized logging, monitoring, and management tools into a unified security monitoring environment.

The architecture diagram provides a high-level view of how network data flows through the platform.

---

# Deployment Environment

Network Sentinel is deployed on:

- Ubuntu Server (Server02)
- Docker Container Platform
- Docker Compose for service deployment and management

Each component runs as an independent container while working together as part of the overall monitoring platform.

---

# Data Sources

Network Sentinel collects security information from monitored network activity.

Primary data sources include:

- Internet traffic
- Router/firewall activity
- Network devices
- Local system events

These data sources provide the information required for detecting suspicious activity and monitoring network behavior.

---

# Security and Logging Pipeline

The security logging pipeline is responsible for collecting, storing, and displaying security events.

## Suricata

Suricata acts as the intrusion detection system (IDS). It analyzes network traffic and generates security events.

Suricata outputs events in JSON format through:

```text
eve.json
```

The `eve.json` file contains security events generated by Suricata, including network activity, alerts, and threat detection information. This file acts as the bridge between the intrusion detection system and the centralized logging pipeline.

## Promtail

Promtail collects log files generated by Suricata and forwards them to Loki.

## Loki

Loki provides centralized log storage and indexing. It allows security events to be searched and queried efficiently.

## Grafana

Grafana provides visualization dashboards that allow users to review security events, analyze trends, and understand detected activity.

# Metrics Collection

Network Sentinel also monitors system performance and container health.

Components include:

## Prometheus

Prometheus collects and stores system metrics.

## Node Exporter

Node Exporter provides host-level metrics including:

- CPU usage
- Memory usage
- Disk activity
- System information

## cAdvisor

cAdvisor provides container-level metrics including:

- Docker container performance
- Resource utilization
- Container health information

# Management and Monitoring

The platform includes management tools to simplify administration.

## Portainer

Portainer provides a graphical interface for managing Docker containers.

## Uptime Kuma

Uptime Kuma monitors service availability and alerts when services become unavailable.

## Homepage

Homepage provides a centralized dashboard for accessing Network Sentinel services.

# Design Philosophy

Network Sentinel was designed as a modular platform where individual services can be deployed, maintained, and expanded independently.

This approach allows future additions such as:

- AI-assisted alert explanations
- Additional security data sources
- Expanded dashboards
- Automated reporting capabilities

The architecture balances functionality and simplicity to remain practical for small organizations and home environments.
